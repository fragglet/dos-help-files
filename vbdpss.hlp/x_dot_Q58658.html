 VB for MS-DOS Quick Libraries Do Not Use Expanded Memory - Q58658
 
 When using a Quick library in VBDOS.EXE, (the programming environment
 included with the Standard and Professional Editions of Microsoft
 Visual Basic version 1.0 for MS-DOS), the Quick library is always
 placed in conventional memory. If you have expanded memory (as
 defined in the Lotus/Intel/Microsoft [LIM] version 4.0 Expanded
 Memory Specification [EMS]), VBDOS.EXE utilizes this expanded memory
 for loaded source code segments that are smaller than 16K; however,
 Quick libraries are always placed in conventional memory.
