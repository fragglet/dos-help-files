Read or Ignore State File (/TSF) - CodeView
                                             ◄Up► ◄Contents► ◄Index► ◄Back►
────────────────────────────────────────────────────────────────────────────
 
     The Toggle State File (/TSF) option either reads or ignores
     CodeView's state file (CURRENT.STS), depending on the setting of
     the Statefileread switch in the CodeView section of TOOLS.INI.
     See: ◄Configuring CodeView in TOOLS.INI►
 
     If Statefileread is set to yes, the /TSF option ignores
     CURRENT.STS. This means that CodeView does not restore the
     breakpoints, watchpoints, and window configurations from the
     previous session.
 
     If Statefileread is set to no, the /TSF option reads the state
     file and sets up CodeView the way it was during the last debugging
     session.
