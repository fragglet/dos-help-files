─────Set Case Sensitivity On (/C)───────────────────────────────────────────
 
     Syntax:  /C
 
     This option indicates that context strings in the source file are
     case sensitive. When the application searches for context strings
     in the help database, case is significant.
 
     For example, a help source file might have the following context-
     string definition for a topic:
 
          .context Open
          topic text for Open
          ∙ ∙ ∙
 
     If this source file is encoded with the /C option, the topic is
     displayed only when the user requests help for "Open." A request
     for help for "open" or "OPEN" would not find this topic.
