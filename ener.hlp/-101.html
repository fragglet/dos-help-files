Bad QLB calling convention
■ The current stack value is incorrect.
 
■ Possible solutions:
 
  • Compare the DECLARE statement for the SUB or FUNCTION
    procedure with the CALL statement to make sure the
    argument list has the same number of items in both cases.
 
  • If you are calling a MASM subroutine, verify that stack
    values are correctly reset.
 
■ If you need additional help, move the cursor to a keyword
  and press F1 to open the Help window.
