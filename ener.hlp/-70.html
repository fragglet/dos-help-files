Permission denied (ERR: 70)
■ More than one condition can cause this error:
 
  • Attempting to write to a write-protected disk.
 
  • Attempting to access a locked file.
 
  • Attempting to use UNLOCK on a file that was not
    previously locked.
 
■ Possible solutions:
 
  • Remove write-protection from the disk.
 
  • Unlock the file before attempting access.
 
  • Lock the file before trying to unlock it.
 
  • Remove the UNLOCK statement.
 
  • Insert error checking to test for disk/file status
    before attempting WRITE or UNLOCK operations.
 
■ If you need additional help, move the cursor to a keyword
  and press F1 to open the Help window.
