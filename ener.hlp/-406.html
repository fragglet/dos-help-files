Blank lines not allowed before SUB/FUNCTION line.
■ Ordinarily, the SUB or FUNCTION statement that begins a
  procedure is the first line in the code window.
 
■ When you try to use the smart editor to put a blank line
  above the SUB or FUNCTION statement, the editor always
  transforms that blank line into a comment line.
 
■ If you need additional help, move the cursor to a keyword
  and press F1 to open the Help window.
