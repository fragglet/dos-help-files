     BIND error U1263
 
     error during link of file, link error status : 'status'
 
     A linking error occurred during the LINK session invoked by BIND.
 
     The following are possible causes of this error:
 
        ■ Unresolved references exist in the files. BIND could not
          resolve references with API.LIB or other support libraries.
 
        ■ APILMR.OBJ was used when the executable file was created, and
          LINK gave error L2044, "symbol multiply defined, use /NOE".
          Relink using the LINK option /NOE, then rebind.
 
        ■ There was not enough memory.
 
        ■ A disk I/O error occurred.
