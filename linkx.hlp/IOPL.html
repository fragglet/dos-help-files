─────<iopl> Attribute───────────────────────────────────────────────────────
 
     Used by:  CODE, DATA, SEGMENTS
 
     Sets whether a segment requires I/O privilege. For OS/2 only.
 
     Keyword     Purpose
 
     IOPL        Specifies that a code segment requires I/O privilege
                 and that a data segment can be accessed only from an
                 IOPL code segment.
 
     NOIOPL      Default. Specifies that there is no I/O privilege
                 required for code and no protection for data.
