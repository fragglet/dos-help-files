 Main Module
 
 ■ In Visual Basic, the main module is the module in a project that:
   • Contains the starting point where processing begins
   • Names the root portion of a project (.MAK)
   • Names the default executable (.EXE) file
 
 ■ One module within each project is identified as the main module. Either a
   form module or code module can be the main module. By default, the first
   module loaded is the main module in the Visual Basic programming
   environment. You can overwrite this selection by choosing Set Start-up
   File from the Run menu.
 
 ■ When the main module file specified is a Basic source code (.BAS) file,
   the project starting point is at the first line of executable module-level
   code. When the main module is a form (.FRM) file, the project starts by
   loading that form file and waiting for any defined form events to occur.
