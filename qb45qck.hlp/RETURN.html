RETURN - a control flow statement that returns control from a subroutine
 
Syntax
  RETURN [{linelabel | linenumber}]
    ■ If the argument is omitted, RETURN continues execution where an event
      occurred (for event handling) or at the statement following the GOSUB
      (for subroutine calls)
    ■ linelabel or linenumber, if used, must identify a line in the
      module-level code to execute next
