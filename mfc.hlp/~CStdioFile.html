  virtual ~CStdioFile();
 
  Remarks
 
  Usually, this destructor closes the operating-system file associated
  with the CStdioFile object. However, if the CStdioFile object was
  constructed by passing in a handle to an opened file using CStdioFile(
  int ), the operating-system file will not be closed. You must close the
  operating-system file yourself.
