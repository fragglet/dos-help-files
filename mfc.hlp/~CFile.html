  virtual ~CFile();
 
  Remarks
 
  Normally, the destructor closes the operating system file associated
  with the CFile object. However, if the CFile object was constructed by
  passing in a handle to an open file using CFile(<int>), the operating
  system file will not be closed. You must close the operating system file
  yourself.
