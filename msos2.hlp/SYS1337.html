SYS1337
 
Explanation
 
A disk error was encountered while CHKDSK with the /F option was attempting
to update the File Allocation Table on the specified drive. The table number
will be 1 or 2, depending on which of the two copies of the File Allocation
Table could not be written.
 
Action
 
If this message appears for both table 1 and 2, format the disk. If FORMAT
fails, the disk is unusable.
 
See Also
 
CHKDSK, FORMAT
