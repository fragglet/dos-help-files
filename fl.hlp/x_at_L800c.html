─────Multiple Threads and DLL Support (/M)──────────────────────────────────
 
     The /M option supports dynamic-link and multithread applications
     in OS/2. These options cannot be used with the /AM option since
     threads and dynamic-link libraries are not supported in medium
     model.
 
     Option     Use
 
     /MT        Multithread applications
 
     /MD        Dynamic-link library and multithread applications
 
     The /MT option automatically links to LLIBFMT.LIB. With /MD all
     default libraries are overridden and you must create and link to
     your own library.
 
     Both the /MT and /MD option imply the /G2 (286 instruction set)
     option, since multithread and dynamic-link library programs run
     only under OS/2. They also imply the /FPi87 (math coprocessor)
     option.
 
     Both /MT and /MD are incompatible with /Fb and /Lr or /Lc, since
     threaded and dynamic-link library applications are assumed to be
     unbound programs running in protected mode.
