     FORTRAN compilation error F2737
 
     <iooption> : lowercase in string nonstandard
 
     If the /4Ys compiler option is used (or the $STRICT metacommand
     is in effect), the value of the specified I/O option must be given
     in uppercase. For example, ACCESS='DIRECT' is legal in this case,
     but ACCESS='direct' is not.
