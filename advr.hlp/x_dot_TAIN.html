 [form.]ctlname.TabIndex[ = index%]
 
 Usage Notes
   ■ Assigning a TabIndex value of less than 0 generates an error.
 
   ■ By default, Visual Basic assigns a tab order to the controls in the
     order in which you draw them on a form. See: ◄Tab Order Definition►
 
   ■ TabIndex for all controls on a form are always unique and consecutive
     (from 0 to n-1) within that form.
 
   ■ If you change the value of a control's TabIndex property to adjust the
     default tab order, Visual Basic automatically renumbers the TabIndex of
     the other controls to reflect insertions and deletions.
 
   ■ A form cannot get the focus if any of its children are able to receive
     the focus. In contrast, a picture box can receive the focus even when it
     contains child controls.
