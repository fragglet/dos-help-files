PWB Error
 
     Not enough memory below 640K: aborting dependency lists generation
 
     PWB ran out of conventional memory while generating dependencies.
 
     Close some files in PWB and try again. If this fails, make more
     memory available to PWB and try again.
 
     If the previous attempts fail, try the following procedure to
     regenerate dependencies:
 
       1. Exit PWB.
       2. Restart PWB with the /DA option.
       3. Open the project.
       4. From the Project menu, choose Edit Project.
       5. Choose Save List.
 
     If this fails, your project is too large for PWB to handle with
     your current hardware configuration. You can continue using the
     makefile as a non-PWB makefile.
