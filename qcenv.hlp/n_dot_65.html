QuickC displays this message if you have edited the file
but have not saved your changes. You need not save the
changes to build a new version of the program. You must
save before you exit QuickC if you wish to maintain the
latest version of your source code on disk.
 
To avoid confusion, save your work as you go along. Saving
your work frequently is also a good idea in case serious
errors or system crashes occur.
 
Choose <Yes> to save the changes before proceeding with
the command.
 
Choose <No> to discard the changes before proceeding
with the command.
