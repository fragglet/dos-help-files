─────Exceeding Code/Data Padding────────────────────────────────────────────
 
     The LINK /PADC and /PADD options add filler bytes to the end of a
     module. This padding serves as a buffer zone so a module can grow
     in size without requiring a full link.
 
     See: ◄Pad Code Segments (/PADC)► (in LINK Help)
          ◄Pad Data Segments (/PADD)► (in LINK Help)
 
     When a module outgrows its padding, an incremental violation
     usually occurs. However, when only one module contributes to a
     segment, the entire segment is moved to avoid an incremental
     violation.
