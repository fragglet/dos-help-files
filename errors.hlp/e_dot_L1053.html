     Fatal LINK error L1053
 
     out of memory for symbol table
 
     The program had more symbolic information (such as public,
     external, segment, group, class, and file names) than could fit in
     available memory.
 
     Try freeing memory by linking from the DOS command level instead
     of from a MAKE file or the QuickC environment. Otherwise, combine
     modules or segments and try to eliminate as many public symbols
     as possible.
