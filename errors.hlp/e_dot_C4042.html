     Compiler warning (level 1) C4042
 
     'identifier' : has bad storage class
 
     The storage class specified for <identifier> cannot be used in
     this context.
 
     The default storage class for this context was used in place of
     the illegal class:
 
        ■ If <identifier> was a function, the compiler assumed extern
          class.
 
        ■ If <identifier> was a formal parameter or local variable, the
          compiler assumed auto class.
 
        ■ If <identifier> was a global variable, the compiler assumed
          the variable was declared with no storage class.
