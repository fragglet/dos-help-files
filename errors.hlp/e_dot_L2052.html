     LINK error L2052
 
     'symbol' : unresolved external - possible calling convention
     mismatch
 
     A symbol was declared to be external in one or more modules, but
     the linker could not find it publicly defined in any module or
     library.
 
     The name of the unresolved external symbol is given, then a list
     of object modules which contain references to this symbol is
     given. The error message and the list are written to the map file,
     if one exists.
 
     This error occurs when a prototype for an externally defined
     function is omitted from a program that is compiled with the
     Fastcall option (/Ox). The calling convention for fastcall does
     not match the assumptions that are made when a prototype is not
     included for an external function.
 
     Either include a prototype for the function, or compile without
     the /Ox option.
