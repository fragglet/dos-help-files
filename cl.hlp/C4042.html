     Compiler warning (level 1) C4042
 
     'identifier' : has bad storage class
 
     The storage class specified for the identifier could not be used in
     this context.
 
     A default storage class for this context was used in place of
     the illegal storage class. The storage class was selected using
     the following rules.
 
        ■ If the identifier was a function, the compiler assumed
          extern class.
 
        ■ If the identifier was a formal parameter or local variable,
          the compiler assumed auto class.
 
        ■ If the identifier was a global variable, the compiler
          assumed the variable was declared with no storage class.
