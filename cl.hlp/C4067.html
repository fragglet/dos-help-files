     Compiler warning (level 1) C4067
 
     unexpected characters following 'directive' directive - newline
     expected
 
     Extra characters followed a preprocessor directive and were
     ignored. This warning appears only when compiling with the /Za
     ANSI-compatibility option.
 
     For example, the following code generates this warning:
 
          #endif    NO_EXT_KEYS
 
     To remove the warning, compile with /Ze or use comment
     delimiters, as in:
 
          #endif    /* NO_EXT_KEYS */
