     Compiler error C2002
 
     invalid wide-character constant
 
     The use of the multibyte-character constant was not legal.
 
     This error can be caused if a wide character contains more bytes
     than expected.
 
     Wide characters cannot be concatenated with ordinary string
     literals and cannot be used if the standard header, STDDEF.H,
     is not included.
 
     Wide-character strings and constants must be preceded by the
     character L.
 
     The following example shows a wide-character constant:
 
          L'mbconst'
 
     Wide characters are not supported in C.
