Conflicts Between __fastcall and Windows Entry/Exit Code
                                             ◄Up► ◄Contents► ◄Index► ◄Back►
─────C/C++ Compiler─────────────────────────────────────────────────────────
 
     Windows entry/exit code and the __fastcall calling convention can
     conflict in their use of the ax register. As a result, using the
     following combinations of options and keywords will generate an
     error:
 
        ■ __fastcall, __far, Gw
 
        ■ __fastcall, __far, __export, GA
 
        ■ __fastcall, __far, __export, GD
 
        ■ __fastcall, __far, GA, GEf
 
        ■ __fastcall, __far, GD, GEf
 
        ■ __fastcall, __far, __export, GA, GEf
 
        ■ __fastcall, __far, __export, GD, GEf
